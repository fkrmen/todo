答辩老师可能会问的问题和回答

1. 你这个项目是做什么的？
回答：
这是一个待办事项管理系统，主要解决用户记录任务、设置优先级、管理截止时间和接收提醒的问题。同时系统还提供了管理员后台，用于管理用户和公告。

2. 为什么选择做这个项目？
回答：
因为待办系统场景比较典型，能完整覆盖前端页面开发、后端接口设计、数据库建模、用户权限控制以及 Linux 部署，适合作为一次综合性的课程设计。

3. 你的项目技术栈是什么？
回答：
前端使用 Vue 3、Vue Router、Element Plus、Axios 和 Vite。
后端使用 Spring Boot、MyBatis-Plus、JWT 和 JavaMail。
数据库使用 MySQL 8.0。
部署环境使用 Linux，后端以 jar 包形式运行。

4. 为什么前端选择 Vue 3？
回答：
因为 Vue 3 上手效率高，组件化开发清晰，适合中小型管理系统开发。再配合 Vue Router 做页面切换，配合 Element Plus 可以快速搭建后台和表单界面。

5. 为什么后端选择 Spring Boot？
回答：
Spring Boot 开发效率高，生态完善，适合快速构建 RESTful API。它对配置管理、依赖整合、定时任务和邮件服务支持都比较成熟。

6. 为什么数据库选择 MySQL？
回答：
因为 MySQL 是关系型数据库里非常常用的一种，和 Java Web 项目集成成熟，适合存储用户、待办事项和公告这类结构化数据。

7. 你的数据库设计了哪些表？
回答：
主要有三张核心表。
第一张是 `fab_user`，保存用户账号、密码、角色、状态、邮箱、昵称等信息。
第二张是 `fab_todo`，保存待办标题、优先级、截止时间、完成状态和提醒状态。
第三张是 `fab_notice`，保存系统公告。

8. 为什么 `fab_todo` 要关联 `fab_user`？
回答：
因为每条待办事项都必须属于某个用户，所以我通过外键把待办和用户关联起来。这样既能保证数据一致性，也方便按用户查询自己的任务列表。

9. 你的项目是前后端分离吗？
回答：
是的，开发阶段是前后端分离。前端通过 Axios 调用后端接口，后端提供 RESTful API。部署阶段我把前端打包后的静态资源放进 Spring Boot 的 `static` 目录，由后端统一通过 8080 端口提供服务。

10. JWT 是做什么用的？
回答：
JWT 主要用于用户登录认证。用户登录成功后，后端生成 token，前端保存 token。之后访问需要权限的接口时，把 token 放到请求头里，后端通过拦截器统一校验。

11. 为什么要用 JWT，而不是传统 Session？
回答：
因为 JWT 更适合前后端分离项目。它是无状态认证，不需要服务器额外保存会话信息，接口调用时只要校验 token 即可，结构更清晰，也便于后续扩展。

12. 你的权限控制是怎么做的？
回答：
后端通过 `JwtInterceptor` 拦截请求，校验 token 后把用户 id、用户名和角色写入请求上下文。普通用户接口只允许访问自己的数据，管理员接口会额外判断角色是不是 `ADMIN`。

13. 管理员和普通用户有什么区别？
回答：
普通用户主要使用待办和个人中心功能。
管理员除了登录系统之外，还可以进入独立后台，对用户进行启用、禁用、重置密码、强制下线，还可以发布和管理公告。

14. 你的项目亮点是什么？
回答：
我认为有三个亮点。
第一是完整性比较强，既有前端页面，也有后端接口、数据库和 Linux 部署。
第二是不只是基础 CRUD，还做了 JWT 权限控制和管理员后台。
第三是加入了邮件提醒和定时任务，让项目更贴近实际应用。

15. 邮件提醒是怎么实现的？
回答：
后端使用 Spring 的定时任务，每隔 5 分钟扫描一次待办事项。如果某条任务满足提醒条件，并且用户已经配置了邮箱，系统就会调用邮件服务发送提醒，并把任务标记为已提醒，避免重复发送。

16. 定时任务为什么有必要？
回答：
因为如果没有定时任务，系统只能被动等待用户自己查看页面。加入定时提醒以后，系统就具备了主动服务能力，可以在任务接近截止时自动通知用户。

17. 你在 Linux 上是怎么部署这个项目的？
回答：
大致分四步。
第一步导入数据库：`mysql -u root -p < db/schema.sql`
第二步构建前端：`npm install` 和 `npm run build`
第三步打包后端：`mvn -DskipTests package`
第四步运行 jar 包，或者交给 `systemd` 管理。

18. 为什么要使用 Linux 部署？
回答：
因为 Linux 更适合服务器运行环境，稳定性好，而且便于通过命令行完成构建、启动、日志排查和服务管理。对于 Java Web 项目来说，这是一种比较常见的生产环境方案。

19. `systemd` 有什么作用？
回答：
`systemd` 可以把 Java 项目注册成系统服务。这样就可以使用 `systemctl start`、`stop`、`restart`、`status` 来管理项目，还能支持开机自启和异常自动重启，运行会更稳定。

20. 如果项目启动失败，你怎么排查？
回答：
我会先看服务状态，再看日志。
例如先执行 `systemctl status fab-todo`，再执行 `journalctl -u fab-todo -f`。
如果是数据库问题，就检查 MySQL 是否启动、账号密码是否正确。
如果是端口问题，就检查 8080 是否被占用。

21. 你的项目接口是怎么组织的？
回答：
我按模块划分接口。
例如用户相关接口放在 `/api/user`，待办相关接口放在 `/api/todo`，管理员相关接口放在 `/api/admin`，公告相关接口放在 `/api/notice`。这样路径清晰，便于维护。

22. 为什么使用 MyBatis-Plus？
回答：
因为它能减少很多基础数据库操作代码，比如单表增删改查可以直接复用框架能力，这样我就可以把精力更多放在业务逻辑和接口设计上。

23. 你这个项目遇到过什么问题？
回答：
一个比较典型的问题是前端打包后后台页面出现白屏。后来我通过调整 Vite 的分包策略，稳定拆分依赖，解决了 Element Plus 相关资源加载问题。

24. 你这个项目还有哪些可以继续优化的地方？
回答：
可以继续从四个方向优化。
第一，增加分页、搜索、标签分类等功能。
第二，引入 Redis 做缓存或验证码控制。
第三，把邮件提醒做得更细，比如支持提前 1 天或 1 小时自定义提醒。
第四，增加单元测试和接口测试，提高工程质量。

25. 如果让你再做一版，你最想优化什么？
回答：
我会优先优化提醒系统和管理员后台。
提醒系统可以支持多种提醒策略，管理员后台可以增加统计图表，比如任务完成率、用户活跃度和公告阅读情况，这样项目会更完整。

26. 这个项目最能体现你掌握了什么能力？
回答：
最能体现的是我具备完整开发链路的能力，也就是从前端页面、后端接口、数据库设计，到 Linux 部署和服务管理，我都能独立完成，而不是只停留在单一模块开发。
