                    Todo Master —— 5 分钟演讲脚本
                    基于 Linux 环境的课程设计答辩

【开场白 —— 约 30 秒】

大家好，我们小组做的是一个前后端分离的待办事项管理系统，叫 Todo Master。

简单来说，它是一个可以帮助用户管理日常任务、设置优先级和截止时间，并且能通过邮件
自动提醒的工具。同时它还有一个完整的后台管理系统，管理员可以对用户和公告进行管理。

整个项目从开发、调试到最终部署，全部基于 Linux 环境完成。接下来我从技术栈、
Linux 使用、核心架构三个方面来介绍。


【一、技术栈 —— 约 1 分钟】

我们采用的是目前业界主流的前后端分离架构：

前端这块：
  - 用 Vue 3.4 框架，它是组合式 API（Composition API）的写法
  - 构建工具用的是 Vite 5，开发时热更新非常快
  - UI 组件库选了 Element Plus，提供了表格、表单、对话框等现成组件
  - 路由用 Vue Router 4，页面按需懒加载，减小首屏体积
  - HTTP 请求用 Axios，统一封装了请求拦截器和响应拦截器

后端这块：
  - 基于 Spring Boot 4.0.6 框架，Java 17
  - ORM 层用的是 MyBatis-Plus 3.5.9，相比原生 MyBatis 省去了大量 XML 编写
  - 认证和鉴权是基于 JWT（JSON Web Token）实现的，无状态，服务端不需要存 session
  - 数据库用的是 MySQL 8.0，字符集用 utf8mb4，支持 emoji 等特殊字符

前后端之间通过 RESTful API 通信。开发时前端跑在 3000 端口，通过 Vite 的 proxy
把 /api 请求代理到后端的 8080 端口，避免了跨域问题。


【二、Linux 环境与命令 —— 约 1 分半钟（重点）】

接下来是 Linux 相关的部分，这也是我们课程设计的核心要求。

首先，整个开发环境是搭建在 Linux 服务器上的。我们用的是 Amazon Corretto 17 这个
JDK 发行版，路径是 /usr/lib/jvm/java-17-amazon-corretto/。

(1) 数据库初始化 —— 用到了经典的 MySQL 命令行操作：

    mysql -u root -p < db/schema.sql

    这条命令把建库建表的 SQL 脚本通过重定向导入 MySQL。schema.sql 里包括了
    CREATE DATABASE、USE、CREATE TABLE 等完整的 DDL 语句。

(2) 项目构建 —— Maven 命令行打包：

    开发阶段我们直接在服务器上用 Maven 启动：
    mvn spring-boot:run

    但更重要的是生产部署时的打包命令：
    mvn -DskipTests package

    这会在 target 目录下生成一个 fat JAR，也就是 fab-todo.jar。这个 JAR 包
    内嵌了 Tomcat，所以不需要额外安装 Web 服务器。

(3) 服务管理 —— systemd：

    这是 Linux 课程的重点内容。我们编写了一个 systemd 服务单元文件
    fab-todo.service，放在了 /etc/systemd/system/ 下。

    关键配置项包括：
    - Type=simple：以简单进程方式运行
    - ExecStart：指定 Java -jar 命令启动
    - Restart=always + RestartSec=10：崩溃后 10 秒自动重启
    - WantedBy=multi-user.target：多用户模式下自动启动

    对应的操作命令：
    systemctl stop fab-todo    —— 停止服务
    systemctl start fab-todo   —— 启动服务
    systemctl status fab-todo  —— 查看运行状态
    systemctl enable fab-todo  —— 设置开机自启

    这些都是我们在 Linux 课程中学到的 systemd 服务管理命令。

(4) 部署验证 —— 用 curl 命令：

    前端打包后嵌入 JAR 包中，由 Spring Boot 作为静态资源提供服务。
    部署完成后我们用 curl 验证：
    curl http://127.0.0.1:8080/ | head -20
    检查返回的 HTML 是否已经是最新版本。

    curl -I http://127.0.0.1:8080/assets/index-xxx.js
    检查静态资源的 HTTP 响应头，确认文件能正常访问。

(5) 前端构建 —— npm 命令：

    cd /opt/todo-project/front
    npm install    —— 安装依赖
    npm run build  —— 生产构建，输出到 dist 目录

    整个项目部署在 /opt/todo-project/ 目录下，这是 Linux 标准的
    第三方软件安装路径。


【三、核心架构设计 —— 约 1 分钟】

(1) JWT 认证拦截器

    这是后端安全的核心。我们实现了一个 JwtInterceptor 拦截所有 /api/** 的请求。
    它把接口分成了三类：
    - 白名单：/api/user/login、/api/user/register、/api/admin/login
      这些接口不拦截，直接放行
    - 可选认证：/api/todo/list、/api/notice/latest
      用于支持首页免登录浏览
    - 强制认证：其他所有接口，不带有效 Token 就返回 401

    Token 过期时间是 24 小时（86400000 毫秒），由 application.yml 配置。

    另外还实现了"强制下线"机制：管理员在后台把用户的 force_logout_at 字段更新后，
    该用户的 Token 即使还没过期，也会被拦截器拒绝。

(2) 跨域 CORS 配置

    因为前后端分离，不同端口会有跨域问题。我们在 WebMvcConfig 中全局配置了 CORS
    过滤器，允许所有来源、所有请求头、所有 HTTP 方法，同时支持携带 Cookie。

(3) 定时邮件提醒

    用 @EnableScheduling + @Scheduled 注解实现。每 5 分钟（300000 毫秒）扫描一次
    数据库，找出截止时间已到但还没提醒过的任务，通过 QQ 邮箱的 SMTP 服务器发送
    提醒邮件给用户。

(4) 前端路由守卫

    在 Vue Router 的 beforeEach 钩子中实现了：
    - 需要登录的页面自动跳转登录页
    - 需要管理员权限的页面检查 role 字段
    - 已登录用户访问登录页时自动跳转到对应首页

(5) Element Plus 分包优化

    前端打包时通过 Vite 的 manualChunks 配置，把 Element Plus 相关组件单独打成一个
    vendor 包。这样做的好处是：Element Plus 体积大但很少变动，单独分包后利用浏览器
    缓存，用户后续访问时不需要重新下载。


【四、项目亮点与难点 —— 约 40 秒】

第一个难点是 Element Plus 的循环依赖导致后台页面白屏。根本原因是 Vite 分包时
不同 chunk 之间产生了循环引用。我们的解决方案是在 vite.config.js 中手动配置
manualChunks，把 Element Plus、Vue、Vue Router、Axios 等分别拆成独立的 chunk。

第二个是在 Linux 上编写 systemd 服务单元。需要理解 Unit、Service、Install 三个
段落的含义，以及 Type、ExecStart、Restart 等参数的作用。

第三个是强制下线功能：需要在 JWT 无状态认证的基础上，增加一个时间戳判断来
实现"让某个 Token 失效"的能力，这是 JWT 本身不具备的。


【五、总结 —— 约 20 秒】

总结一下，Todo Master 是一个完整的前后端分离项目，涵盖了：
- 前端 Vue 3 + Vite + Element Plus 全家桶
- 后端 Spring Boot + MyBatis-Plus + JWT
- MySQL 数据库设计（外键约束、索引优化）
- Linux 环境下使用 systemd 管理服务、Maven 打包部署、curl 验证
- 以及邮件提醒、权限控制、强制下线等进阶功能

以上就是我们小组的课程设计项目，谢谢大家！
