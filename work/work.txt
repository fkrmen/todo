Todo Master 项目说明

1. 项目概述

Todo Master 是一个前后端分离的待办事项管理系统。前端使用 Vue 3、Vite、Element Plus，后端使用 Spring Boot、MyBatis-Plus、JWT 和 MySQL。

当前交付版本已经包含以下能力：
- 首页允许未登录浏览，只有真正执行添加任务等操作时才跳转登录
- 待办新增区改为“手动添加主入口 + 智能解析辅助入口”
- 新增一句话智能解析待办，自动提取标题、优先级、截止时间
- 截止时间改为快捷时间胶囊和自定义时间两段式交互
- 首页徽标按时间显示问候语，例如“上午好，张三”
- 个人中心支持独立维护 QQ、昵称、邮箱、密码
- 管理后台支持用户管理、公告管理、强制下线、密码重置
- 系统支持邮件提醒

2. 技术栈

- 前端：Vue 3、Vue Router 4、Element Plus、Axios、Vite 5
- 后端：Spring Boot 4、MyBatis-Plus 3.5.9、JWT、JavaMail
- 数据库：MySQL 8.0
- Java：17

3. 主要功能

用户侧：
- 用户注册、登录
- 待办新增、编辑、删除、完成状态切换
- 手动填写标题、优先级、截止时间直接创建任务
- 一句话智能解析，辅助生成待办草稿
- 截止时间快捷选择：今天结束前、明天、3天内、一周内、清空
- 个人中心维护 QQ、昵称、邮箱、密码
- 公告查看与已读状态记录
- 到期邮件提醒

管理员侧：
- 管理员独立登录页
- 用户启用/禁用
- 用户密码重置
- 用户强制下线
- 公告发布、编辑、删除

4. 智能解析能力

后端新增接口：
- `POST /api/todo/parse`

请求体示例：
- `{ "text": "明天下午3点提交周报，高优先级" }`

返回体中的 `data` 包含：
- `title`
- `level`，取值为 `LOW / MEDIUM / HIGH`
- `deadline`，格式为 `YYYY-MM-DDTHH:mm:ss`，无法确定时返回 `null`

兜底规则：
- 标题解析失败时，回退为原始输入句子
- 优先级解析失败时，回退为 `MEDIUM`
- 截止时间解析失败时，回退为 `null`

5. 运行方式

数据库初始化：
mysql -u root -p < db/schema.sql

启动后端：
cd back
mvnw.cmd spring-boot:run

Linux / macOS 可使用：
mvn spring-boot:run

启动前端：
cd front
npm install
npm run dev

默认地址：
- 前端开发环境：http://localhost:3000
- 后端接口：http://localhost:8080

6. 生产部署说明

更新前端后，如需部署到 8080 提供静态资源，必须执行：
- `cd front && npm run build`
- `cd ../back && mvnw.cmd -DskipTests package`

后端 `pom.xml` 已将 `front/dist` 打包进 JAR 的 `static/` 目录，只重启旧 JAR 不会自动更新前端资源。
