Todo Master 更新部署说明

适用场景：
- 已部署旧版本
- 现在需要把前端界面、后端逻辑、数据库脚本和交付材料一起更新到当前版本

一、本次更新重点

需要同步的主要变化：
- 待办新增区改为“手动添加主入口 + 智能解析辅助入口”
- 新增一句话智能解析接口 `POST /api/todo/parse`
- 截止时间 UI 改为快捷时间胶囊 + 自定义时间
- 首页 Hero Badge 改为按时间显示问候语
- `work/` 目录中的源码镜像、配置镜像、数据库脚本和说明文档已同步更新

二、本地开发运行

1. 初始化数据库
mysql -u root -p < db/schema.sql

2. 启动后端
cd back
mvnw.cmd spring-boot:run

3. 启动前端
cd front
npm install
npm run dev

说明：
- 前端默认端口是 3000
- Vite 会把 `/api` 代理到 `http://localhost:8080`
- 如果页面报 `ECONNREFUSED`，通常表示后端没有启动

三、生产服务器更新流程

1. 停止旧服务
systemctl stop fab-todo

2. 上传最新代码和交付材料
需要至少同步以下目录或文件：
- `back/`
- `front/`
- `db/schema.sql`
- `work/`

3. 重建前端
cd /opt/todo-project/front
npm install
npm run build

4. 重打包后端
cd /opt/todo-project/back
mvn -DskipTests package

5. 启动服务
systemctl start fab-todo
systemctl status fab-todo

6. 验证服务是否已经加载新版本
curl http://127.0.0.1:8080/

四、DeepSeek 配置说明

智能解析由后端直连 DeepSeek，推荐配置为环境变量或独立密钥管理：

- `DEEPSEEK_BASE_URL`
- `DEEPSEEK_API_KEY`
- `DEEPSEEK_MODEL`
- `DEEPSEEK_TIMEOUT_MS`

如果缺少 `api-key` 或 `model`，`/api/todo/parse` 会返回 `503`。

五、数据库说明

本次功能没有新增数据库表结构，仍使用现有待办字段：
- `title`
- `level`
- `deadline`

因此：
- 全新环境可直接导入 `db/schema.sql`
- 旧环境通常不需要因为智能解析单独改表

六、建议核对项

更新完成后建议至少验证以下场景：
- 手动填写标题后可直接创建任务
- 智能解析可以回填标题、优先级、截止时间
- 截止时间快捷胶囊和自定义时间都能正常工作
- 首页已登录状态显示“上午好/下午好/晚上好，用户名”
- 未登录时首页仍可浏览，但添加任务会跳转登录

七、补充

- Windows 本地开发优先使用 `mvnw.cmd`
- `work/配置文件/fab-todo.service` 适用于 Linux `systemd` 部署
- 每次前端改动后，都要重新执行前端构建再打包后端，否则 8080 仍可能提供旧静态资源
